## 升级机制分析（本次会话回答用户问题）

### 内机（室内机）升级方式

**入口：** `main.c` 中调用流程：
```
upgrade_file_check_decompression()   // 检测并解压 TF 卡固件包
upgrade_check_firmware()             // 检测各模块并执行升级
```

**固件识别：`access()` 检测文件是否存在**
- 固件包放在 TF 卡：`/mnt/tf/firmware`（tar 包）
- 解压到：`/tmp/upgrade/`（`FIRMWARE_PATH`）

**升级判断逻辑（`upgrade_check_firmware` 中）：**
用 `platform_mask`、`rings_mask`、`app_mask` 等 bitmask 判断哪些文件需要升级：
- 如果存在 `usr.jffs2`（mask 0x40），特殊处理：先备份涂鸦 ID，goto platform 跳过其他检测
- 依次检测：platform 文件、铃声 mp3、壁纸、ANYKA37E.BIN、rom.bin、sat_leo.ttf

**各升级分区（用 `mtd_debug erase/write` 直写 NOR Flash）：**
| 文件 | MTD 分区 | 说明 |
|------|---------|------|
| env_ak3761e_nor.img | mtd1, mtd2 | ENV/ENVBK |
| EVB_CBDM_AK3760E_V1.0.1.dtb | mtd3 | DTB |
| uImage | mtd4 | KERNEL |
| anyka_logo.rgb | mtd5 | LOGO |
| root.sqsh4 | mtd6 | ROOTFS |
| usr.jffs2 | mtd7 | CONFIG |
| usr.sqsh4 | mtd8 | APP |

**app/资源文件升级方式（非 MTD 分区）：**
cp 到 `/etc/config/run/` 目录：
- `ANYKA37E.BIN`（主程序）
- `rom.bin`（UI 资源）
- `sat_leo.ttf`（字体）
- 铃声、壁纸 jpg

**升级完成后：** 屏幕红白闪烁（无限循环），等待人工重启。
**注意：** `upgrade_file_check_decompression` 当前有 `return;` 直接返回，实际上 TF 卡自动解压功能被禁用；但 `upgrade_check_firmware` 仍会检测 `/tmp/upgrade/` 目录。

### 门口机（outdoor）远程升级

文件：`network_outdoor_upgrade.c`
- 升级包路径：`/mnt/tf/net_camera.up`（先 cp 到 /tmp 再打开）
- 命令：`NET_COMON_CMD_UPGRADE_OUTDOOR (0x61)`
- 流程：分包发送（每包 1510 字节），`arg1` = 包序号，`arg2` = 数据长度，发完后再发 `arg2=1` 表示结束
- 进度条通过 LVGL bar 显示
